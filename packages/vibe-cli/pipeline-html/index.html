7-Stage Design Pipeline

seed prompt elicitor.md bdd.feature *.tla / *.cfg impl plan
1

Elicitor

Interactive

Opens a new terminal for a freeform interview. One question at a time until all design dimensions are resolved.

In Seed prompt
Out docs/<feature>/elicitor.md
elicitor.md 50 turns
2

BDD Writer

Transforms the elicitor briefing into Gherkin BDD scenarios.

In elicitor.md
Out bdd.feature
bdd-writer.md
3

BDD Debate

Loop

Expert panel debates the BDD scenarios against the original briefing. On consensus, the recommendation is applied by the writer.

Briefingelicitor.md
Artifactbdd.feature
Reference—
Debate Moderator
PARTIAL_CONSENSUS
BDD Writer revises
CONSENSUS_REACHED BDD Writer applies recommendation
debate-moderator.md + bdd-writer.md 100 rounds
4

TLA+ Writer + TLC

Loop

Writes a TLA+ formal specification from the BDD scenarios, then runs the TLC model checker. Loops until the spec passes verification.

In bdd.feature
Out tla/*.tla, *.cfg
TLA+ Writer
TLC Model Checker java -jar tla2tools.jar
errors
pass Spec verified
tla-writer.md 100 attempts
5

TLA+ Debate

Loop

Expert panel reviews the TLA+ spec against the elicitor briefing, using the BDD scenarios as reference to verify alignment.

Briefingelicitor.md
Artifact*.tla
Referencebdd.feature
Debate Moderator
PARTIAL_CONSENSUS
TLA+ Writer revises
CONSENSUS_REACHED TLA+ Writer applies recommendation
debate-moderator.md + tla-writer.md 100 rounds
6

Implementation Writer

Produces a detailed implementation plan with task breakdown, execution tiers, and a JSON manifest for the coding stage.

In *.tla
Out implementation-plan.md + .json
implementation-writer.md
7

Implementation Debate

Loop

Expert panel reviews the implementation plan against the elicitor briefing, using the TLA+ spec as reference for state/transition coverage.
