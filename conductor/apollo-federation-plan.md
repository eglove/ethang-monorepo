# Implementation Plan: Apollo Federation with ethang-graphql

## Objective
Create a new Apollo Federation gateway (`ethang-graphql`) running on Cloudflare Workers and migrate the existing `ethang-rss` app to a schema-first Apollo Subgraph to support Federation.

## Key Files & Context
- `apps/ethang-graphql/` (New directory)
  - `package.json`, `wrangler.jsonc`, `src/index.ts`, `src/supergraph.graphql`
- `apps/ethang-rss/`
  - `package.json`
  - `src/index.ts`
  - `src/graphql/schema.ts` -> Will be removed/replaced.
  - `src/graphql/schema.graphql` -> New schema file.
  - `src/graphql/resolvers.ts` -> New resolvers file.
- Root Configuration
  - `package.json` (Add Rover CLI for local supergraph composition)

## Implementation Steps

### Phase 1: Convert `ethang-rss` to an Apollo Subgraph
1. **Dependencies:**
   - Remove `drizzle-graphql`.
   - Install `@apollo/subgraph` and `graphql-tag` in `ethang-rss`.
2. **Schema-First Approach:**
   - Create `src/graphql/schema.graphql` with Federation v2 directives (`@key`, etc.) and the types previously generated by `drizzle-graphql` (e.g., `Subscription`, `Mutation.addSubscription`, `Query.subscriptions`).
3. **Resolvers:**
   - Create `src/graphql/resolvers.ts`.
   - Write manual resolvers using Drizzle ORM to match the schema, preserving the logic from `add-subscription.ts` and `subscriptions.ts`.
4. **Server Configuration:**
   - Update `src/index.ts` to use `buildSubgraphSchema({ typeDefs, resolvers })` from `@apollo/subgraph`.

### Phase 2: Create `ethang-graphql` Gateway
1. **Scaffold App:**
   - Create the `apps/ethang-graphql` directory.
   - Initialize `package.json` and `wrangler.jsonc`.
2. **Dependencies:**
   - Install `@apollo/server`, `@apollo/gateway`, `graphql`, and `@as-integrations/cloudflare-workers`.
3. **Gateway Implementation:**
   - Create `src/index.ts`.
   - Setup `ApolloGateway` referencing the local `supergraph.graphql` file (or using a supergraph string/URL format compatible with Cloudflare Workers since local file reads are restricted).
   - *Note:* Since Cloudflare Workers cannot easily read local files dynamically, we will import the supergraph schema as a string or use a build step to inline it.
   - Setup `ApolloServer` with the gateway and `startServerAndCreateCloudflareWorkersHandler`.

### Phase 3: Supergraph Composition (Rover CLI)
1. **Rover Setup:**
   - Install `@apollo/rover` as a dev dependency in the root workspace or advise the user to use the global CLI.
   - Create a `supergraph.yaml` file in `apps/ethang-graphql/` defining the `ethang-rss` subgraph and its routing URL.
2. **Build Scripts:**
   - Add a script in the root `package.json` or `apps/ethang-graphql/package.json` to compose the supergraph: `rover supergraph compose --config supergraph.yaml > src/supergraph.graphql`.

## Verification & Testing
1. Compose the supergraph using Rover.
2. Run `ethang-rss` locally (`pnpm dev`).
3. Run `ethang-graphql` locally (`pnpm dev`).
4. Execute test queries against `ethang-graphql` (e.g., fetching `subscriptions` or adding a subscription) and ensure it correctly routes to and resolves from `ethang-rss`.
