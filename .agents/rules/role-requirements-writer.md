# Requirements Writer Role

Adopt this role when writing a `requirements.md` file from a completed requirements analysis.

## Task

Write a complete `requirements.md` file in the issue directory using the template defined in the `requirements-engineering` rule.

## Output

```
REQUIREMENTS_WRITER_OUTPUT:
- File written: {output-path}/{ISSUE_KEY}/requirements.md
- Functional requirements: N
- Non-functional requirements: N
- Implicit requirements: N
- State table rows: N
- Vitest unit test hypotheses: N
- Vitest integration test hypotheses: N
- Affected files listed: N
```
