# Tool Usage Hierarchy

This repository strictly enforces a hierarchy of tools for all AI agents. Native execution of arbitrary scripts or fallback tools must be avoided if specialized tools exist.

## 1. Primary Toolset: codebase-memory-mcp
The codebase-memory-mcp is the TOP PRIORITY for everything it supports. You must use its tools whenever applicable.
- `search_graph`
- `trace_path`
- `get_code_snippet`
- `query_graph`
- `get_architecture`

## 2. Secondary Toolset: webstorm-mcp
For anything not covered by codebase-memory-mcp, use webstorm-mcp.
This includes:
- Writing, creating, and renaming files
- Refactoring codebase elements
- Project builds and execution
- Database searches and SQL queries

## 3. Fallback: PowerShell & Specialized CLIs
If the MCP tools cannot fulfill the requirement, use PowerShell as a fallback.
**BASH IS STRICTLY BANNED.** Do not use bash for any operations.
Within PowerShell, leverage available specialized CLIs:
- `/ripgrep` (rg) for searching text (Preferred when searching outside the repo)
- `/jq` for processing and reading JSON files
- `/everything-search` (es) for fast file lookups (Preferred when searching outside the repo)
- `/github-cli` (gh) for git management
- `/sara-cli` for tracing requirements

## 4. Banned Native Tools
The following native actions are restricted by a PreToolUse hook and must be redirected to their webstorm-mcp, codebase-memory-mcp, or CLI equivalents:
- `run_command` (if invoking bash or native read/edit instead of an MCP)
- `replace_file_content` / `multi_replace_file_content` / `write_to_file` (must use webstorm-mcp for writes/edits if possible)
- `view_file` / `read_file` (must use codebase-memory-mcp or webstorm-mcp)
- `list_dir` (use webstorm-mcp list_directory_tree)
- `find_by_name` (use webstorm-mcp find_files_by_name_keyword or /everything-search)
- `grep_search` (use codebase-memory-mcp, webstorm-mcp search tools, or /ripgrep)
