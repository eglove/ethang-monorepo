JavaScript: The Standard Library

One of the most requested features for JavaScript is for a standard library. But what does that mean? Isn't everything on the global window object a part of a "standard library"?

JavaScript provides some things like Math, Date, JSON, Intl, and fetch. But these are relatively limited compared to languages like Python, Java, and C#.

What would JavaScript need to qualify as "having a standard library"? I don't think there is a definitive answer to this. But below is a set of standard, mature libraries that can help make up for where JS is lacking.

What makes up a standard library

  • Standard Error Handling: An ergonomic way to handle errors such as via types Result<T>, typed returns, or a build time awareness of uncaught throws.
  • Data Manipulation : Advanced data structures such as Map, Set, Queue, Graphs, Trees, etc., as well as common utilities to handle deep copying, sorting collections efficiently, etc.
  • Time and Date Handling : Full support for data and time with timezones as well as utilities to handle calendars.
  • Networking: Standardized APIs for sockets, WebSockets, and HTTP.
  • Concurrency/Asynchrony: High-level abstractions for managing async workflows. (event emitters, task queues, worker threads)
  • String and Regular Expression Utilities: Built-in utilities for efficient manipulation of strings or pattern matching. Standardized functions for natural language processing (tokenization, stemming, etc.)
  • Web APIs: Common utilities for URL parsing, query strings, encoding/decoding structured data types.
  • Mathematics and Randomness: Support for working with vectors, matrices, and general numeric computing. As well as PRNG utilities for reproducible randomness.
  • File I/O: Methods to write files directly, available for browser environments and server-side. The browser File System API has limited user interaction, limited scope, limited file metadata, no support for watching directories, batch file management, symlinks, file locks, or any standard API to handle remote files. (FTP, HTTP, S3, etc.). The node fs module has inconsistent platform handling, higher level abstractions for temporary directories, atomic writes and retry mechanism are missing. It also has no standard interface to handle remote files and no built-in compression.
  • Cryptography: Standardized, secure solution for hashing, encryption, and public/private key operations.
  • OS Level Control: Access to environment variables, parallel threads (with robust abstractions), process management.

The Libraries

Lodash: Safely and elegantly handle JS type coercion without throws. Functional error handling. Deep object cloning. Collection handling. Functional methods such as flow (pipe pattern). Complex string manipulation such as casing and HTML escape.

Luxon: (Temporal is still experimental). As this article says, "The major issues with JavaScript's Date object are that it only supports the user's local time and UTC, and there's no time zone support. Additionally, its parsing behavior is very unreliable, and Date itself is mutable, which can introduce hard-to-trace bugs. There are other problems like calculations across Daylight Saving Time (DST) and historical calendar changes, which are notoriously difficult to work with."

TanStack Query: Asynchronous state management, retries, queues, deduplication, error handling, invalidation, etc.

TanStack Table: Table data state management and utilities.

TanStack Form: Form state management and utilities.

TanStack Store: Generic reactive state management.

qs: The native URL and URLSearchParams objects are great. But they cannot handle nested objects, arrays, serialization, and provide no utility for relative paths.

cookie: Manipulating cookie headers often involves custom string parsing, combining headers, deduplication, and normalization.

data-structure-typed: Provides typed interfaces and performant implementations of advanced data structures like queues, graphs, trees, etc.

CortexJS Compute Engine: Complex math including constants such as ExponentialE and MachineEpsilon, calculus expressions, imaginary and complex number support, numeric domains, mathematical functions, linear algebra support, statistics support, trigonometry support.

Hyper Fetch: Standardized APIs for data exchange including sockets and handling caching, queueing, offline support, and persistence.

ZenFS: Isomorphic virtual file system with standardized API using various backends. (In-memory, IndexedDB, localStorage, etc.)

threads.js: Isomorphic way to manage parallel tasks with worker threads.

natural: Natural language processing for Bayesian and Logistic Regression Classifiers, Entropy, Sentiment Analysis, etc.
