Why You Should Install That JS Library

One of the most requested features for JavaScript is for a standard library. But what does that mean? Isn't everything on the global window object a part of a "standard library"?

JavaScript provides some things like Math, Date, JSON, Intl, and fetch. But these are relatively limited compared to languages like Python, Java, and C#.

What would JavaScript need to qualify as "having a standard library"? I don't think there is a definitive answer to this. But below is a set of standard, mature libraries that can help make up for where JS is lacking.

Feature Vanilla Library
Build Time Features None TypeScript, adds build-time type checking, enums, decorators, and namespaces.
Linting None EsLint, broad support and ecosystem
Standard Formatting None Prettier, opinionated formatting standards
Standard Error Handling Error objects do not qualify lodash attempt/isError
Typed Throws JS has no awareness or build time metadata or what throws. No known definite solutions.
Advanced Data Structures Very limited (Map/Set) Well-supported by data-structure-typed
Data Structure Utilities Inconsistent interfaces for objects, arrays, strings, etc. Generally throws on error. Generic "collection" methods provided by lodash. Generally uses graceful fallbacks on error.
Time/Date Handling Date does not support timezones, provide calendar utilities, unreliable parsing behaviors, mutable. (See Temporal blog.) Luxon, standard interface with timezone support, format parsing, and calendar utilities.
DOM Manipulation Very basic, imperative methods React (most popular), reactive and declarative.
Styling CSS, robust but has not evolved with the web. Contradicts component driven design. TailwindCSS, uses modern best practices like component-driven design, strips unused styles for performance, adds themeing support for consistent design systems.
Cookies Strings only from headers or document.cookie no parsing or utilities. cookie, handles parsing, combining headers, deduplication, and normalization.
Reactive state management (Observers) None TanStack Store, generic state management with framework adapters.
Forms HTTP forms, limited validation, inconsistent API from AJAX and JSON TanStack Form, complex validation, state management, supports both Form and AJAX.
Table Data Handling None TanStack Table, provides generic API's for filtering, ordering, sorting, etc. on table data.
Networking and Async Inconsistent interfaces for fetch, sockets, etc., no common utilities. TanStack Query/Hyper Fetch, standard interface for network handling, caching, polling, sockets, data storage, prefetching, deduplication, error/retry handling, network status, offline handling, queues, etc.
Concurrency Worker allows scripts to run in background, send messages. threads.js, High level, isomorphic way to use Workers (observables, pooling, queues, serialization, etc.)
Natural Language Processing None natural, Bayesian and Logistic Regression Classifiers, Entropy, Sentiment Analysis, etc.
RegEx Simple, limited support for RegEx testing Libraries for building complex regex patterns are limited and not well-supported.
URL Parsing Basic utilities for URL and SearchParams. No support for paths (beyond new URL('path/1', https://example.com')), nested objects, arrays or serialization. qs, Full support for complex URL management.
Mathematics Very limited, poor floating point accuracy. Cortex Compute Engine, mathematical constants, imaginary and complex number support, numeric domains, mathematical functions, utilities for linear algebra, statistics, and trigonometry.
Pseudo Random Number Generation Does not exist Some small libraries, no major support
File read/write (browser and server) File System Api has limited user interaction, limited scope, limited metadata, no watch support, no batch file management support, no symlinks, no file locks. Node fs has inconsistent platform handling, poor high level abstractions, no atomic writes, no retry mechanism, no compression. Zen FS, isomorphic virtual file system with standardized API for various backends. (In-memory, IndexedDB, localStorage, etc.)
Standard remote file API (FTP, HTTP, S3, etc.) Does not exist Zen FS, includes interfaces for working with Cloud)
Cryptography Basic support for encryption and public/private key operations. No major popular solutions.
Process Management Node has very basic spawn, exec, and fork methods. Nothing available for browsers. PM2, process and cluster management. Made specifically to manage Node deployments.
String manipulation Treats strings as enumerable, good Internationalization support, a few basic utilities like toLowerCase() and toUpperCase() lodash, more complex casing methods, HTML escaping and templating.
