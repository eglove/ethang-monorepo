# Phase 5 Release and Pull Request Automation

## 1. Description
This use case covers how the \`/sdlc-5\` skill automates the release process for completed feature work by checking branch safety, staging changes, creating a conventional/actor-action formatted commit based on diff analysis, pushing to remote, and opening a Pull Request using the GitHub CLI.

## 2. Actors and Stakeholders
- **AI Release Agent**: The primary actor executing \`/sdlc-5\` to stage, commit, and release the changes.
- **Lead Developer**: Confirms branch switching/creation, reviews commit messages, and merges the resulting pull request.

## 3. Flow of Events
1. The AI Release Agent is invoked with \`/sdlc-5\`.
2. The agent validates the current branch name against protected branches (e.g. \`master\`, \`main\`).
3. If on a protected branch, the agent prompts the developer for a new branch name, creates it, and checks it out.
4. The agent inspects git status to ensure changes exist.
5. The agent stages all changes.
6. The agent performs a git diff analysis to identify modified packages, files, and purposes.
7. The agent generates a conventional commit message following the actor-action formatting rules.
8. The agent commits the staged files with the generated message.
9. The agent pushes the branch to the remote repository.
10. The agent runs \`gh pr create\` to create a new Pull Request.
