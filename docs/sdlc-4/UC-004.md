# Phase 4 Verification and Testing

## 1. Description
This use case covers how the `/sdlc-4` skill executes the verification loop (lint, test with coverage, build), analyzes coverage data and logic states for gaps, and automatically writes missing tests and applies TDD fixes.

## 2. Actors and Stakeholders
- **AI Verification Agent**: The primary actor executing `/sdlc-4` to verify the codebase.
- **Lead Developer**: Receives the final verification status and resolves issues if the agent halts.

## 3. Flow of Events
1. The AI Verification Agent is invoked with `/sdlc-4 <feature-name>`.
2. The agent runs SARA graph validation (`rtk sara check`) to confirm baseline documentation integrity.
3. The agent executes a compile, lint, and test cycle with Vitest coverage enabled.
4. If lint errors are found, the agent runs up to 3 rounds of ESLint autofixes.
5. The agent parses the Vitest coverage output.
6. The agent performs static analysis of the feature's source code to discover state space gaps.
7. If gaps are found, the agent automatically writes corresponding tests using TDD discipline.
8. The agent runs a final validation and checks SARA integrity again.
