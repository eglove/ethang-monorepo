# Guided Implementation & Development

## 1. Description
This use case covers how the `/sdlc-3` skill guides an AI developer through implementing a designed feature while enforcing Phase 3 rules.

## 2. Actors and Stakeholders
- **AI Developer Agent**: The primary actor executing `/sdlc-3` to implement a feature.
- **Lead Developer**: Reviewer of the code and any unresolved linting errors.

## 3. Flow of Events
1. The AI Developer Agent is invoked with `/sdlc-3 <feature-name>`.
2. The agent verifies the SARA requirements and design documents under `docs/<feature-name>/` using `sara check`.
3. If the graph is invalid, the agent halts and reports errors.
4. If valid, the agent executes the TDD cycle:
   - Writes a failing test.
   - Implements minimal code to make it green.
   - Refactors the code.
5. The agent runs linter checks.
6. If lint errors are found, the agent runs up to 3 rounds of autofixes.
7. If errors persist after 3 rounds, the agent halts and hands off to the user.
